# Security Policy

## Supported Versions

We are committed to ensuring the security of the application. We provide security updates for the latest version. Please ensure you are running the most recent version to receive security patches.

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |

## Reporting a Vulnerability

We take all security vulnerabilities seriously. If you discover a security issue, we appreciate your help in disclosing it to us in a responsible manner.

To report a security vulnerability, please use the **private vulnerability reporting feature** on GitHub. You can submit a report by going to the ["Security" tab of the repository and clicking "Report a vulnerability"](https://github.com/SaiSivarajuIN/ai_think/security/advisories/new).

This ensures the vulnerability is disclosed privately to the project maintainers. Please do **not** report security vulnerabilities through public GitHub issues.

We will do our best to acknowledge your report within 48 hours and will work with you to understand and resolve the issue.

Thank you for helping keep this project and its users safe.

## Security best practices

- Regularly updating to the latest version of AI Think
- Securing access to hosted instances of AI Think
- Monitoring systems for unusual activity

## Data and Third-Party Services

This application is designed to be self-hosted and does not collect or transmit any personal data to the project maintainers.

However, the application integrates with third-party services, such as Ollama and optionally Langfuse and ChromaDB. Your interactions with these services are subject to their respective privacy policies and terms of service. We recommend you review the privacy policies of any third-party services you configure for use with this application.

- **Ollama**: Your prompts and generated content may be sent to your configured Ollama instance.
- **Langfuse (Optional)**: If you enable Langfuse integration, usage data, prompts, and responses will be sent to your Langfuse instance.
- **ChromaDB (Optional)**: If you configure ChromaDB Cloud, your chat history and settings will be stored in your ChromaDB Cloud instance.
