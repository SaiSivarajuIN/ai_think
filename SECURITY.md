# Security Policy

This document outlines security procedures and general policies for the AI Think project.

## Supported Versions

We provide security updates for the latest version of the application. Please ensure you are running the most recent release to receive security patches.

## Reporting a Vulnerability

The AI Think team and community take all security vulnerabilities seriously. We appreciate your efforts to responsibly disclose your findings, and will make every effort to acknowledge your contributions.

We use **GitHub Security Advisories** for the responsible disclosure of security vulnerabilities. This ensures that your report is handled privately and securely.

To report a security vulnerability, please go to the **"Security" tab** of our GitHub repository and click on **"Report a vulnerability"**.

Provide a clear and concise description of the vulnerability, including:
- The component or feature where the vulnerability exists.
- Steps to reproduce the issue.
- The potential impact of the vulnerability.

Our team will acknowledge your report and work with you to understand and resolve the issue. We will keep you informed of our progress and will publicly disclose the vulnerability once a patch is available.

**Please do not open public issues for security vulnerabilities.**
